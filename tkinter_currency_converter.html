Written Response

The purpose of my program is to convert currencies into their foriegn values. The program is seen taking multiple different currency inputs and converting them into other currencies,as well as logging past inputs in a list box. The input of the program is a float value representing a value of one of four different currencies. The output is three float values representing the conversion to the remaining three currencies.

List





The list box is labeled “Past Currency Inputs” on-screen, and is called “list_past” in the code. The data in the listbox represents numerical values of currency previously input by the user. I wanted to make some way for users to remember past numbers they’ve input so they can experiment and make comparisons more easily.


Algorithm






The identified program takes the currency input from the user and logs it in a listbox. This helps the user experiment with different conversions without having to remember their past inputs.


Procedure

1. Type an input into the calculator using the on-screen buttons

2. Push the on-screen “Convert” button

3. The number you input will be cleared from the calculator and logged in the “Past inputs” listbox.


On line 135, the function “convertCAN()” calls the procedure to update the “past inputs” listbox along with 3 functions that convert Canadian dollars into USD, Euros, and Japanese Yen. On line 151 the function “convertYEN()” calls the procedure to update the “Past Inputs” listbox along with three functions to convert Japanese Yen into USD, CAN, and EUR.


Conditions Tested For

1. If the calculator is set to Canadian Dollars and if the user has pushed the “Convert” button.

2. If the user has set the calculator to take Japanese yen input. If the user has pushed the “Convert Button”
